# motion-references/browser-zooms/

**Built component:** [`../../../components/browser-frame`](../../../components/browser-frame)

## What is this?
Clips of **browser / UI window reveals + slow zooms** you like — a screenshot rising into a rounded
window then a gentle push-in over the content. The reference feel; the built `browser-frame`
component is the implementation (slides up + scales in, then a slow Ken Burns).

## When to use it?
Pull when building or tuning a UI-proof beat where a screenshot or dashboard fills a browser frame.
Match the scale-in and the Ken Burns speed to the clips here.

## What feeling?
Premium product-shot energy — the UI is presented, not thrown. Slow, deliberate, never frozen.

### What I like (target bullets)
- Window slides up + scales in, fast in / slow settle, no bounce.
- Slow Ken Burns push-in on the content — always moving, never static, never fast.
- Content is top-anchored (object-fit cover) so the header reads first.
- Muted window chrome; the single coral accent is the tiny secure dot, nothing more.
- Clean hold / gentle drift, then a light-flare or spotlight hand-off.
- No parallax, no tilt, no 3D — flat, on-brand.
