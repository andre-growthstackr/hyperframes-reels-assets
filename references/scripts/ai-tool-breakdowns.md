# ai-tool-breakdowns

**What is this?** Spoken phrasing for walking through an AI tool or workflow on screen — the
register for "here's the tool, here's what it does, here's the output." Grouped by the beat of a
breakdown.

**When to use it?** Videos built on `browser-frame` / `ui-highlight` / `claude-prompt-box` where you
demo a tool. Pull lines to narrate the screen without hype.

**What feeling?** Calm, digital, "look how simple this is." You're the operator who's already figured
it out and is showing the shortcut. Never breathless about the tech.

---

## Block A — Naming the tool without pitching it
*When to use:* introduce the tool. State what it does, not how amazing it is.

- Det här är verktyget. Det gör en sak, och det gör den bra.
- Jag använder det här för att slippa redigera för hand.
- Kort sagt: rå video in, färdig reel ut.
- Det ersätter inte dig. Det tar bort det tråkiga.

## Block B — The input
*When to use:* show what goes in. Keep it concrete.

- Jag laddar upp den råa videon. Det är hela jobbet.
- Ingen mall, ingen timeline. Bara klippet.
- Du ger den din inspelning som den är.

## Block C — What happens
*When to use:* narrate the automatic part. This is the payoff beat.

- Sen sköts resten. Captions, zoom, musik.
- Den lägger captions vid rätt ord automatiskt.
- Zoomen sätts där det behövs. Du gör ingenting.
- Bakgrundsmusiken läggs på i rätt läge.

## Block D — The output
*When to use:* reveal the finished result on screen.

- Och det här kommer ut. Färdigt att lägga upp.
- En render, sen är det klart.
- Samma klipp anpassat för fyra plattformar.

## Block E — The honest limit
*When to use:* build trust by naming what it doesn't do. Denial-pivot in miniature.

- Det gör inte allt. Men det gör det tråkiga.
- Manuset skriver du fortfarande själv. Med flit.
- Det är inte magi. Det är bara rätt steg automatiserade.
