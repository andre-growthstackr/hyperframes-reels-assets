# motion-references/transitions/

**Built component:** [`../../../components/light-flare-transition`](../../../components/light-flare-transition)
(the signature move) — plus the pack's default **hard cut + Reveal** for plate entries (see
[`../../../frame.md`](../../../frame.md) → *Motion & Timing*).

## What is this?
Clips of **general scene-to-scene moves** you like — the broader transition vocabulary beyond the
signature flare. Hard cuts, Reveal entries, gentle scale hand-offs. The `light-flares/` folder is
the specific signature; this folder is the wider grammar (and the boundary of what's *not* allowed).

## When to use it?
Pull when deciding how any two scenes connect. Default is a **hard cut + Reveal**; a real *section*
change earns a `light-flare`. This folder collects references for both, and — importantly — **bad
examples** of transitions the brand bans.

## What feeling?
Clean, calm, intentional. Cuts are crisp; the one soft move is the warm flare. Nothing flashy.

### What I like (target bullets)
- Hard cut between plates is the default — no crossfade on identity/claim beats.
- Reveal entry: focal element fades up + rises ~1.2cqw, ease-out-expo, ~0.5s; chrome follows 80–120ms later.
- The one soft transition is the warm light-flare, reserved for real section changes (3–6 per short).
- Fast in, slow settle everywhere; no whip, no spin, no zoom-blur.

### Bad examples to label (the boundary)
- Hard full-frame white flash, glitch, whip-pan, dark wipe, star-wipe — all banned; keep clips of
  these tagged *bad* so the AI learns the edge.

Drop clips + a `notes.md` (shape in the parent README). `.gitkeep` holds the folder until then.
