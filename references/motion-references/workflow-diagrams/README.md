# motion-references/workflow-diagrams/

**Built component:** [`../../../components/workflow-diagram`](../../../components/workflow-diagram)

## What is this?
Clips of **node-flow reveals** you like — a source → tool → output diagram assembling itself. The
reference feel; the built `workflow-diagram` component is the implementation (nodes stagger,
connectors draw, coral on the hero/tool node).

## When to use it?
Pull when building or tuning a source-diagram / step-map / "where the data flows" beat. Match the
node stagger and the connector-draw speed to the clips here.

## What feeling?
Legible, deliberate, calm. The flow builds so the eye can follow it, one step at a time.

### What I like (target bullets)
- Nodes stagger in, fast in / slow settle each, left-to-right or top-to-bottom.
- Connectors *draw* between nodes (line grows), they don't just fade in.
- Coral fires on exactly one node — the hero (tool) node — never all.
- No bounce; a small pop on each node is fine.
- Connector draw is synced to the node it connects, not ahead of it.
- Clean hold once assembled, then a light-flare out to the next section.

Drop clips + a `notes.md` (shape in the parent README). `.gitkeep` holds the folder until then.
